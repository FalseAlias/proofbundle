# Contributing

## Provenance and Lineage Rules
- Do not delete historical artifacts.
- Do not edit frozen dated source artifacts.
- Add dated successor artifacts for corrections.
- Preserve explicit lineage in filenames and documentation.

## Claim Hygiene Rules
- Keep repository statements implementation-grounded.
- Do not represent demonstration behavior as operational guarantees.
- State limits and absences explicitly.

## Scope Control Rules
- Keep the single-file browser architecture unless a concrete technical need is documented.
- Avoid introducing package managers, build steps, frameworks, or CI machinery without necessity.

## Pull Request Expectations
- Describe what changed, why, and what stayed intentionally out of scope.
- Identify whether changes touch source artifact, entrypoint, successor artifact, or documentation.
- Preserve non-destructive repository history.

## 2. Method Used to Compute Hashes

### 2.1 Script

The script `compute_manifest.py` (at repository root) performs the following operations:

1. **Tree walking**: Recursively traverses the repository starting from the root directory.
2. **Filtering**: Applies exclusion rules (see Section 3) to skip generated artifacts and dependency directories.
3. **Hash computation**: For each included file, reads the file in 64 KiB chunks and computes:
   - **SHA-256** (`hashlib.sha256`)
   - **SHA-384** (`hashlib.sha384`)
   - **SHA-512** (`hashlib.sha512`)
4. **Sorting**: File entries are sorted by relative path (POSIX format) for deterministic output.
5. **Merkle root**: A binary Merkle tree is built over the sorted SHA-256 digests (see Section 4).
6. **Output**: Writes `manifest.json` with the computed hashes and Merkle root.

### 2.2 Hash Algorithms

| Algorithm | Digest Size | Use Case |
|-----------|-------------|----------|
| SHA-256   | 32 bytes    | Primary integrity check; used for Merkle root computation |
| SHA-384   | 48 bytes    | Medium-term collision resistance; compliance with some standards |
| SHA-512   | 64 bytes    | Maximum collision resistance; defense-in-depth |

All three are NIST-approved hash functions (FIPS 180-4).

### 2.3 Chunked Reading

Files are read in 64 KiB (65536-byte) chunks to handle large files without excessive memory consumption. The same chunk is fed to all three hash objects simultaneously, so only one pass over the file is required.

## 4. Merkle Root Computation Method

The Merkle root is computed over the **sorted** SHA-256 digests of all tracked files.

### 4.1 Algorithm (binary Merkle tree)

```
1. Convert each SHA-256 hex string to its 32-byte binary representation.
2. Sort the 32-byte digests lexicographically (for determinism).
3. Build a binary tree from the bottom up:
   a. Pair adjacent digests: hash(concat(left, right)) using SHA-256.
   b. If an odd number of digests remains, the last digest is promoted
      to the next level unchanged.
   c. Repeat until a single digest remains.
4. Return the final digest as a 64-character hex string.
```

### 4.2 Properties

- **Deterministic**: Same file set always produces the same Merkle root (sorting guarantees this).
- **Collision-resistant**: Relies on SHA-256's collision resistance; finding two different file sets with the same Merkle root requires finding a SHA-256 collision.
- **Incremental verification**: Individual files can be verified without reading the entire manifest by providing a Merkle path.

## 6. Verification

To independently verify the manifest hashes:

```bash
# Verify a single file's SHA-256
sha256sum README.md
# Expected: e7628d850a19a6b3f9b31b22cf21bfb15fba36f367a76c02175d37196b867ea6  README.md

# Re-run the full manifest computation
python3 compute_manifest.py
```

The script is idempotent: running it multiple times on the same file tree produces identical output (assuming no file modifications).

## 8. Reproducibility

To reproduce this manifest on a fresh checkout:

```bash
python3 compute_manifest.py
```

Requirements:
- Python 3.8+
- No external dependencies (uses only `hashlib`, `json`, `os`, `pathlib` from stdlib)

The script is self-contained and uses only the Python standard library, ensuring reproducibility across environments.
